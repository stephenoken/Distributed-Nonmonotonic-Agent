\module{ODProver} %%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%

\label{ODProverUser}

The program {\tt ODProver} is a query answering
prover with an improved (faster) implementation.

This program can be run using the Hugs interpreter,
or compiled with GHC and run directly from the shell.


\submodule{Usage (GHC)} %%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%

Run the program by typing a command of the form:

\begin{alltt}
$ ODProver\it options\tt [\it{theory-file-name}\tt [\it{tagged-literal}\tt]]
\end{alltt}

\noindent The program options, commands and behavior
are the same as described for
{\tt DProver} in section~\ref{DProverUser}, with
the following exceptions:

\begin{itemize}
   \item Prover engines with well-founded sematics
      are not available.
   \item Some additional provers with an array-based history 
      for improved speed are provided.
   \item Run-files are not implemented. Consequently
      there is no {\tt -r} command line option or
      {\tt r} command.
\end{itemize}

\submodule{Usage (Hugs)} %%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%

Load the script {\tt ODProver.lhs} into the Hugs
interpreter. The program should be invoked and
used the same way as {\tt DProver}.
